# Clarification Questions

## 1. Readability
**Requirement:** The system has to check sentence length and its complexity.

- What is the allowed sentence length?
- What is considered a “complex” sentence?
- How can the system detect the end of a sentence if punctuation is missing?

---

## 2. Repetition
**Requirement:** The system should check repeated words.

- How many times must a word be repeated to be considered repetition?
- How can the system detect synonyms with the same meaning but different words?
- Are punctuation marks (commas, periods, question marks, etc.) considered repetition?

---

## 3. Uploading
**Requirement:** The system should allow the user to upload and edit narrative text files.

- Where should the files be stored?
- Should the files be public or private?
- Are we allowed to store these files initially?

---

## 4. Suggestion
**Requirement:** The system has to provide suggestions for correcting weak parts of the text.

- What defines a “weak” part of the text?
- What sources should be used to generate good suggestions?
- What format should the suggestions have for the editor?

---

## 5. Layout of the Text
**Requirement:** The system should organize the text by checking spacing, capitalization, and sentence structure.

- What is an “organized” sentence?
- How can the system detect if spacing was intentional?
- What defines the start of a sentence?

---

## 6. Summary of the Text
**Requirement:** The system should generate a summary of the text with key points.

- What are “key points” (clue points)?
- Should the generated summary be a file or a popup?
- How should the summary feature be triggered (button, notification, etc.)?

---

## 7. Text-to-Speech
**Requirement:** The system should convert text to speech in multiple languages via a button.

- What voice options should be available?
- Should the audio be downloadable or only playable within the system?
- Where should the “play” button appear?

---

## 8. Comments
**Requirement:** The system should allow comments on changed lines.

- Who can add comments?
- Should comments be public or private?
- Should comments be exportable with the document?

---

## 9. Comparison
**Requirement:** The system should allow users to compare old and edited versions.

- What types of changes should be highlighted?
- How should differences be displayed?
- Should differences be color-coded?

---

## 10. Dictionary
**Requirement:** The system should provide explanations for complex words.

- What defines a “complicated” word?
- Should the system support multiple languages for definitions?
- Should synonyms and examples be included?
